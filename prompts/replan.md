# 重规划提示词

## 使用说明
当执行偏差过大、目标变更或外部条件变化时使用。

---

## 提示词

```
你正在执行重规划。

当计划与实际执行偏差过大时，重新调整计划。

触发条件：
1. 任务完成率连续低于 40%
2. 目标 deadline 临近但进度严重落后
3. 用户主动要求调整
4. 外部条件变化（如考试时间变更）
5. 发现原计划不可行

重规划原则：
1. 不推翻重来，而是增量修正
2. 分析偏差原因，记录到失败记忆
3. 调整任务粒度（可能太粗或太细）
4. 调整任务数量（可能太多或太少）
5. 调整优先级排序
6. 保留有效的部分，只改需要改的

重规划流程：
1. 读取当前计划和任务
2. 读取 QuickLog 了解实际执行情况
3. 读取失败记忆
4. 分析偏差原因
5. 生成调整方案
6. 更新相关文件（带变更原因标注）

输出：
1. 重规划记录 → plans/{{weekly|monthly}}/replan-YYYY-MM-DD.md
2. 更新相关任务文件
3. 更新 goals/ 中相关目标
4. 更新 memory/patterns.md（如有新发现）

重规划文件格式：
---
id: replan-YYYY-MM-DD
date: YYYY-MM-DD
trigger: {{触发原因}}
created_at: YYYY-MM-DD

## 偏差分析
- 原计划：{{简述}}
- 实际执行：{{简述}}
- 偏差率：{{百分比}}
- 原因分析：{{为什么偏了}}

## 调整方案
1. {{调整1}} — 原因：{{为什么}}
2. {{调整2}} — 原因：{{为什么}}
3. {{调整3}} — 原因：{{为什么}}

## 保留不变
- {{有效的部分}}

## 失败记忆更新
- {{新发现的模式（如有）}}

## 下一步
- {{接下来该做什么}}
```
