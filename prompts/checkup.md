# 巡检提示词

## 使用说明
定期检查系统健康状况时使用。

---

## 提示词

```
你正在执行 Stone Plan 系统巡检。

检查整个生活管理系统的健康状况。

检查维度：
1. 目标健康：是否有长期无进展的目标？是否有冲突的目标？
2. 任务健康：是否有长期挂在 backlog 里的任务？任务粒度是否合理？
3. 连击健康：哪些连击快断了？断连频率如何？
4. 数据完整性：QuickLog 是否有缺失？健康/财务数据是否最新？
5. 失败记忆：是否有新的重复模式需要记录？
6. 文件一致性：索引文件是否与实际文件同步？
7. 知识库引用：引用的知识库文件是否仍然有效？

巡检规则：
1. 先出报告，不自动修改
2. 每个问题给出证据和修复建议
3. 按优先级分类（高/中/低）

输出：checkups/checkup-YYYY-MM-DD.md

巡检报告格式：
---
id: checkup-YYYY-MM-DD
date: YYYY-MM-DD
created_at: YYYY-MM-DD

## 概览
- 进行中目标：{{数量}}
- 待办任务：{{数量}}
- 活跃连击：{{数量}}
- 发现问题：{{数量}}

## Findings

### High Priority
#### {{问题1}}
- 类型：{{目标停滞 | 任务堆积 | 连击危机 | 数据缺失 | 索引不同步}}
- 证据：{{具体说明}}
- 建议：{{修复方案}}

### Medium Priority
...

### Low Priority
...

## 统计
| 检查项 | 状态 |
|--------|------|
| 目标是否有进展 | ✅/⚠️ |
| 任务粒度是否合理 | ✅/⚠️ |
| 连击是否健康 | ✅/⚠️ |
| 数据是否完整 | ✅/⚠️ |
| 索引是否同步 | ✅/⚠️ |
| 失败记忆是否更新 | ✅/⚠️ |
```
