# TRAE Solo Web 端配置指南

> 本文件说明 Stone Plan 在 TRAE Solo Web 独立端的配置方式。
> TRAE Solo Web 使用 **Skills 系统**，不是智能体也不是 CLI 命令。

---

## 配置方式

Skills 文件已放在 `.trae/skills/` 目录下，共 11 个技能：

```
.trae/skills/
├── stone-plan/SKILL.md    # 主控（会话启动、日常交互）
├── quicklog/SKILL.md      # 一句话快速记录
├── today/SKILL.md         # 今日任务
├── nudge/SKILL.md         # 老爷爷模式
├── goal/SKILL.md          # 目标拆解
├── plan/SKILL.md          # 计划生成
├── review/SKILL.md        # 复盘
├── replan/SKILL.md        # 重规划
├── checkup/SKILL.md       # 系统巡检
├── diet/SKILL.md          # 饮食记录（热量/蛋白质/碳水计算）
└── dashboard/SKILL.md     # 仪表盘管理
```

### 如何让 TRAE 识别这些 Skills

**方式一（推荐）：直接使用**
把 `stone-plan` 文件夹作为 TRAE Solo Web 的工作目录打开，Skills 会自动被识别。

**方式二：手动导入**
1. 打开 TRAE Solo Web → 设置 → 规则和技能
2. 点击"创建" → 上传 SKILL.md 文件或 .zip 包
3. 选择"项目"类型

**方式三：通过对话创建**
直接在 TRAE 对话中说："帮我在 .trae/skills 目录下创建技能"，AI 会自动生成。

| 技能 | 触发关键词 | 用途 |

| Stone Plan 主控 | "今天做什么"、"系统状态"、"我的目标" | 系统入口 |

| 目标拆解 | "我想XX"、"新目标" | 创建/拆解目标 |
| 计划生成 | "周计划"、"月计划" | 生成计划 |
| 复盘 | "复盘"、"总结这周" | 周/月复盘 |
| 重规划 | "调整计划"、"重规划" | 计划偏离时调整 |
| 巡检 | "巡检"、"系统检查" | 系统健康检查 |
| 饮食记录 | "吃了什么"、"这顿多少热量"、发食物图片 | 饮食热量计算 |
| 仪表盘 | "启动仪表盘"、"看面板" | Dashboard 管理 |

## 日常使用

Skills 支持自动触发和手动调用：

```
# 自动触发（直接说自然语言）
"我跑了30分钟"          → 自动调用 QuickLog
"今天做什么"            → 自动调用 今日任务
"下一步"                → 自动调用 老爷爷模式
"我想学吉他"            → 自动调用 目标拆解
"复盘这周"              → 自动调用 复盘

# 手动调用
"用 QuickLog 技能记录一下：花了30块吃午饭"
"用目标拆解技能帮我拆解：考过六级"
```

## 最小启动

如果觉得 8 个太多，**只保留 2 个就够日常使用**：

1. **stone-plan**（主控）— 包含所有规则，日常交互入口
2. **quicklog**（记录）— 最常用的功能

其他功能直接对 stone-plan 说自然语言也能处理。
