# 巡检

## 描述
定期检查整个 Stone Plan 系统的健康状况，发现潜在问题并给出修复建议。

## 使用场景
- 用户说"巡检"、"系统检查"、"健康检查"
- 一周/一月结束时自动建议
- 系统感觉不对劲时手动触发

## 指令

### 启动时必读
1. AGENTS.md
2. goals/_index.md
3. memory/patterns.md
4. knowledge-ref/_ref-index.md

### 检查维度
1. **目标健康**：是否有长期无进展的目标？是否有冲突的目标？deadline 是否临近？
2. **任务健康**：是否有长期未完成的任务？任务粒度是否合理？
3. **连击健康**：哪些连击快断了？断连频率如何？
4. **数据完整性**：QuickLog 是否有缺失？健康/财务数据是否最新？
5. **文件一致性**：索引文件（_index.md）是否与实际文件同步？
6. **知识库引用**：引用的知识库文件是否仍然有效？

### 巡检规则
1. 先出报告，不自动修改
2. 每个问题给出证据和修复建议
3. 按优先级分类（高/中/低）

### 输出
1. 写入 checkups/checkup-YYYY-MM-DD.md
2. 友好展示巡检摘要
3. 如有高优先级问题，主动建议修复
