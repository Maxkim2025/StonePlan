# 失败记忆

> 记录重复失败的模式，帮助 Agent 在生成任务时规避已知风险。
> 最后更新：2026-04-26

---

## 失败模式列表

### pattern-001：计划与执行脱节
- **类型**：plan_miss
- **触发条件**：计划在对话中生成，但没有立即执行第一个任务
- **表现**：计划看起来很完美，但当天完成率为0%
- **频率**：1次
- **最近发生**：2026-04-25
- **应对策略**：生成计划后，立即引导用户执行第一个最简单的任务（如"记录早餐"），降低启动门槛
- **关联目标**：goal-fitness, goal-cet4, goal-ai-interview

### pattern-003：任务数量固定不灵活
- **类型**：plan_miss
- **触发条件**：每天固定排6个任务，不管目标紧迫度和当天是否有训练
- **表现**：考前冲刺期任务量不够，训练日任务又太多
- **频率**：1次
- **最近发生**：2026-04-26
- **应对策略**：任务量由目标紧迫度驱动，不设固定上限。训练日减少学习任务，休息日/有氧日增加学习任务。考前冲刺可以加到10+个任务
- **关联目标**：goal-cet4, goal-ai-interview

### pattern-004：腿臀训练过度疲劳
- **类型**：energy_crash
- **触发条件**：腿臀训练太累，一周两次无法恢复
- **表现**：用户主动要求减少腿训频率
- **频率**：1次
- **最近发生**：2026-04-26
- **应对策略**：腿臀训练改为一周一次，不练腿的力训日改为有氧爬坡
- **关联目标**：goal-fitness, goal-cet4, goal-ai-interview

### pattern-002：任务描述过于笼统
- **类型**：plan_miss
- **触发条件**：任务只写"运动20分钟"而不指定具体内容
- **表现**：执行时不知道该做什么，容易跳过
- **频率**：1次
- **最近发生**：2026-04-25
- **应对策略**：任务描述必须包含具体动作（如"健身房训练Day1：背+肩后束+肱二头，引体/高位下拉/划船6-8组"）
- **关联目标**：goal-fitness

### pattern-005：自动生成丢失训练任务
- **类型**：plan_miss
- **触发条件**：自动生成逻辑在周计划任务提取不足时，用默认模板补充，但默认模板不包含训练任务
- **表现**：训练日（Day2/3/4/6）自动生成的任务里没有训练，变成了和Day1一样的学习任务
- **频率**：1次
- **最近发生**：2026-04-26
- **应对策略**：默认模板需要根据 day_num 区分训练日/有氧日/休息日，训练日必须包含训练任务。用户去健身房训练，不是居家训练
- **关联目标**：goal-fitness
